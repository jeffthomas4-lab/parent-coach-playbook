# Migrations

Cloudflare D1 schema migrations for the **shared activity database** (`activity-radar`).
Forward-only SQL, sequence-prefixed so Wrangler applies them in order.

| File | What it creates |
|---|---|
| `0001_core_graph.sql` | organizations, programs, sessions, activity_categories |
| `0002_trust_and_demand.sql` | trust_signals, accreditations, search_events |
| `0003_operations.sql` | submitters, org_claims, reviews, featured_listings, geocoded_addresses, domain_quality |
| `0004_seed_categories.sql` | controlled-vocabulary activity categories |

The migrated camp data is **seed data, not a schema migration**. It is generated by
`scripts/migrate_camps.py` to `seed/0005_seed_from_camps.sql` and applied with
`wrangler d1 execute --file`, never with `migrations apply`. Keeping it out of this
folder stops the migration runner from re-running it and hitting duplicate-key errors.

## First-time setup (production)

Run once on the Windows dev machine.

```powershell
npx wrangler login
npx wrangler d1 create activity-radar
```

Paste the returned `database_id` into `wrangler.jsonc`. Then:

```powershell
npx wrangler d1 migrations apply activity-radar --remote
```

## Migrating the camps data

```powershell
# 1. Export the old approved camp rows
npx wrangler d1 execute parent-coach-playbook --remote --json --command "SELECT * FROM camps WHERE status='approved';" > camps_export.json

# 2. Generate the seed (dry-run first to read the domain report)
python scripts/migrate_camps.py camps_export.json --dry-run
python scripts/migrate_camps.py camps_export.json --out seed/0005_seed_from_camps.sql

# 3. Apply the seed to the shared DB (execute --file, never migrations apply)
npx wrangler d1 execute activity-radar --remote --file=./seed/0005_seed_from_camps.sql

# 4. Sanity check
npx wrangler d1 execute activity-radar --remote --command "SELECT (SELECT COUNT(*) FROM organizations) AS orgs, (SELECT COUNT(*) FROM programs) AS programs;"
```

Phase 5 (a later session) re-points `parentcoachdesk.com/camps` to read this DB
filtered to `program_type='camp'` by adding a second D1 binding to that repo's
`wrangler.jsonc` pointing at the same `database_id`.
